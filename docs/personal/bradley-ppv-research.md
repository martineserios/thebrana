# August Bradley — PPV (Pillars, Pipelines, Vaults)

**Date:** 2026-03-09
**Purpose:** Input for personal space design (input #3)

---

## Core Structure

**Pillars** — Life areas and values (career, family, health, learning, finances, growth). The foundation everything else builds on. Each pillar has a "north star" direction.

**Pipelines** — Active workflows channeling work toward pillars:
- Value Goals → Goal Outcomes (OKR-like) → Projects → Action Items/Tasks
- Hierarchical: daily actions trace up to life values

**Vaults** — Knowledge storage and retrieval. Captures, organizes, and resurfaces information at the moment you need it.

**Alignment Zone** — The integration point. All pillars, values, goals, projects, and daily tasks connect here. This is the unique contribution — visible traceability from daily work to life purpose.

## Alignment Mechanism

```
North Star (yearly) → Pillars (yearly) → Value Goals + OKRs (quarterly)
  → Projects (weekly) → Daily Actions (automated, pre-prioritized)
```

You know why you're doing today's work because it traces up to what matters most. Actions are pre-prioritized by their goal alignment.

## Review Cadences

| Cadence | Focus |
|---------|-------|
| **Daily** | Execute pre-prioritized actions |
| **Weekly** | Review projects, adjust task priorities |
| **DuoCycles** (monthly/quarterly hybrid) | Review goals and OKRs, recalibrate |
| **Yearly** | Mine past year, review/redefine north star and pillars |

## Notion Implementation

- Heavy use of database relations and rollups for the hierarchy
- Formula fields automate prioritization based on goal alignment
- Templates for recurring reviews
- **Performance warning:** Notion slows with heavy rollup usage

## Strengths

- **Comprehensive life OS** — handles tasks, goals, knowledge, life design in one system
- **Emergence** — parts work together as greater than sum
- **Clear alignment** — daily work visibly connected to life purpose
- **Flexibility** — adapts as life changes

## Weaknesses

- **Complex** — dozens of hours of YouTube content recommended before implementing
- **High maintenance** — breadth creates upkeep overhead
- **Notion-dependent** — performance degrades, no offline mode
- **Learning curve** — not intuitive; requires full architecture understanding first

## vs PARA

| Dimension | PARA | PPV |
|-----------|------|-----|
| Scope | Information organization | Full life operating system |
| Goal setting | Not covered | Core feature (OKRs per pillar) |
| Life design | Not covered | Core feature |
| Task execution | Task-agnostic | Tasks tied to values |
| Complexity | Minimal | Significant |

PPV is more ambitious — manages your entire life, not just documents.

## Patterns for Personal Space

| Pattern | Application | Priority |
|---------|-------------|----------|
| Pillar-based life areas | Define 5-7 life pillars, not just professional | High |
| Alignment zone (daily → purpose) | Every action traces to a goal traces to a pillar | High |
| Review cadences (daily/weekly/quarterly/yearly) | Structured reflection rhythm | High |
| OKRs per pillar | Measurable targets for each life area | Medium |
| DuoCycles (monthly/quarterly hybrid) | More practical than separate monthly + quarterly | Medium |

## What NOT to Steal

- Notion-specific implementation (we're file/markdown-based)
- The full complexity (too much upkeep for solo operator)
- Pre-built templates (build our own aligned to brana conventions)

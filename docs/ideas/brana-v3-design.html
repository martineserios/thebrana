The Graduated Process Loop

Brana v3 in one sentence: you stop operating the system and start designing the loop that operates it. The loop parallelizes, judges, challenges, and verifies work — and every process earns its autonomy step by step. Efficiency is the constraint, simplification is the method, the public platform falls out of the boundary we cut.

The autonomy ladder

The core decision. No loop is "autonomous" by declaration — each process climbs, and your daily cockpit approvals are the evidence that promotes it. Judgment-heavy work is capped at L2 forever, by design.

L1 · REPORT-ONLY

It watches and tells you

Triage summaries, drift reports, extraction digests. No action taken.

Every loop starts here. Zero risk, builds the habit and the data.
L2 · ASSISTED — THE COCKPIT

It prepares, you approve

Candidates generated in parallel, judged, challenged, verified, diffed. Your decision takes seconds.

Where development lives. Each approval is graduation evidence for L3.
L3 · UNATTENDED — EARNED

It acts, you audit receipts

Runs and merges without you — overnight, unattended.

Graduation gate — per task shape, not per task: verifier + encoded stops + the shape clearing a transparent rule table (≥K runs, ≥95% merged-clean, auto-demotion) on real outcome-ledger evidence. First graduate: the LEARN loop.

The four waves

Ordering principle: each wave proves the ladder's next rung while paying for the one after it. ADRs ride inside the wave they gate — there is no spec-only wave.

WAVE 1 · v3.1

Ground truth + Supersession

Diagnose the two claimed failures (stalled extraction queue; stale memory path). Superseding ADR retires the prior autonomous-runner doc cluster — its mechanics are now native to this design. ADR-060 amendment scoped. Native agent memory migration.

DELETES prior runner doc cluster · stale read paths · t-1994/1995
WAVE 2 · v3.1

LEARN autonomous

Async local worker: agy/Gemini first (free), automatic engine-switch to Claude on quota exhaustion (SDK credits / claude -p) — the queue always drains, never defers to a reset. Curation gate + run ceiling inside the wave.

DELETES agy skip-and-defer stall path · close FULL sync-extraction weight
WAVE 3 · v3.1

Verify — extend, don't replace

Verifier extends build-evaluator + validate.sh. goal-completion.sh migrated guard-by-guard (anti-gaming carry-forwards named, tests ported, 10+ consumers migrated). Outcome ledger as ladder state.

DELETES goal-completion.sh — only after consumer migration
WAVE 4 · v3.1

Cockpit + Router

Cost-baseline spike first (N=1 judge-only fallback). Approval = PR merge through the harvested ADR-060 gate chain. Judge = shipped verify-findings workflow. Router profiles built in.

DELETES manual phase invocations · prose routing rules
WAVE 5 · v3.1

Graduate + Cut

Shape-based graduation from the outcome ledger — transparent rule table, auto-demotion, gated on a real soak. First qualifying task shape goes L3 (activates the ADR-060 amendment). Core vs process-packs boundary — the public path (t-2090) inherits it.

DELETES v2 remnants retired against the v3 spec

The wave contract

Non-negotiable, adopted from the pre-mortem — these two rules kill the two most likely failure modes (spec sprawl, cost blowout):

  • ≤10 tasks per wave · each wave ships into daily use · next wave gated on the previous one shipping
  • Each wave deletes more surface than it adds (hooks + skills + rules net-negative)
  • Per-loop token budgets are stop conditions from wave 1 · LEARN runs on a separate compute budget
  • Graduation to L3 requires verifier + stops + N clean supervised runs — never declaration

# NotebookLM Test Prompt

## How to Test

Upload TWO sources to NotebookLM for each test:
1. `system-content.md` (always the same — the Somos Mirada system)
2. One style guide (rotate: style-1-sketch, style-2-blueprint, style-3-minimalist, style-4-darkmode, style-5-organic)

Then use this prompt in the chat:

---

## Prompt

Generate an image of a LinkedIn carousel slide (1080 x 1350 px, portrait orientation) showing a **before/after system diagram** based on the Somos Mirada patient flow described in the sources.

The slide should show:
- **Top section:** Title — "A surgical practice was losing 40% of follow-up patients."
- **Middle section:** Two diagrams — BEFORE (the inherited system, disconnected) and AFTER (the designed system, connected). Use the visual style described in the style guide source. Show the components, connections, and especially the "silent bottleneck" in the before, and the new components (memory layer, human gate, correction loop) in the after.
- **Bottom section:** The phrase "Todo es un sistema." as a closing signature.

Follow the visual style guide exactly — colors, line quality, shapes, mood, and component visual language. The result should feel like a branded, consistent design that could be part of a series.

---

## What to Compare

After generating all 5, compare:
1. Which style is most readable on mobile?
2. Which feels most "you"?
3. Which would a founder find approachable?
4. Which would a technical person respect?
5. Which could you produce consistently — same style, different content, every week?

The winner becomes your visual identity for the content pipeline.

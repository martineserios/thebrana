# Research Discipline

When starting research on any topic:

1. **Read project docs first.** Grep/read the project's own documentation before any web search.
2. **Note what the docs decided** — vocabulary, constraints, decisions, open questions.
3. **Research externally from the doc foundation** — deepen what docs sketched, validate assumptions, find implementations, discover what docs couldn't know.
4. **Cross-reference findings** against doc decisions — do they conflict? extend? answer an open question?

Never launch web research and doc reading in parallel. Web results without doc context produce generic findings instead of project-specific ones.
